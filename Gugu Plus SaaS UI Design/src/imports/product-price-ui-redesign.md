Redesign the interface so it feels warmer, more human, and consumer-friendly — while still remaining minimal, calm, and data-focused.

This is NOT a developer tool, internal admin dashboard, or ops panel.

The product is a consumer-facing price history tracker for international shopping.  
Users track products over time and view price history and insights.

The interface should feel like a well-designed consumer product that happens to show data — not a developer analytics panel.

------------------------------------------------

REMOVE THESE QUALITIES

• Dark developer-style sidebars
• Cold gray system UI palette
• Dense table-first layouts
• Database viewer aesthetics
• Dev-tool or internal admin atmosphere
• Overly rigid grid systems
• Monospace-heavy UI
• Hyper-compact spacing

The current UI should NOT feel like:
Vercel dashboard, Supabase admin, Grafana, or an internal developer console.

------------------------------------------------

BRING IN THESE QUALITIES

Warm visual tone.

Use warm off-whites instead of pure white:

#fafaf8  
#f7f6f3  
#f4f3ef

Cards should feel soft and organic:

• Gentle rounded corners
• Subtle shadows
• Slight elevation
• Soft layering instead of hard borders

Typography:

Use a humanist sans-serif that feels editorial and friendly.

Avoid terminal-like or system-ui feeling fonts.

The typography should feel designed and intentional.

Numbers and prices should remain prominent, but presented with hierarchy and elegance — not dumped into dense data tables.

Accent color:

Use a subtle warm accent.

Examples:
• Soft teal
• Sage green
• Muted warm blue

Avoid overly “functional SaaS teal”.

Spacing:

Generous whitespace.

Data should feel curated, calm, and breathable — not packed or overwhelming.

Interaction details:

• Gentle hover states
• Smooth micro-transitions
• Thoughtful iconography with balanced weight
• Buttons that feel soft and tactile

------------------------------------------------

OVERALL FEEL

The product should feel like:

A beautifully designed consumer product that happens to show data.

Notion-level calmness  
Robinhood-style clarity  
Well-crafted indie SaaS aesthetics

A casual international shopper should feel comfortable exploring this interface.

It should feel welcoming and human — not technical or intimidating.

------------------------------------------------

REFERENCE PRODUCTS / DESIGN INSPIRATION

Instead of Linear-style developer UI, take inspiration from:

• Moneybird
• Lunchclub
• Early Cron calendar
• Notion
• Robinhood

These products balance warmth, whitespace, and thoughtful typography while still presenting structured information.

------------------------------------------------

GOAL

Create a minimal but warm data interface where product price information feels curated, elegant, and approachable for everyday shoppers — not developers analyzing logs.
